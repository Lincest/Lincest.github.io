实际上 GPU 的进程执行是一种 「轮转调度」 的方式, 每个进程的 context 在不断被切换, 每个时刻只有一个进程在 GPU 上执行, 无法达到「并行」, 并且由于上下文切换的开销, 吞吐率反而下降.

Hyper-Q (queue)

Hyper-Q 是一个需要 硬件支持 的特性, 允许多个 CPU 线程或者进程同时 加载任务 到一个 GPU 上, 实现 CUDA kernels 的 并发执行

Hyper Q 支持的连接类型

  • Multi cuda streams
  • Multi cpu threads
  • Multi cpu processes (MPS)
