「编译原理」期末复习

0 | 考点

from 张南老师

编译期末试卷题型

一、填空题 20分，每空1分

二、简答题 20分，共4~5题

三、计算题 60分，共4题

考试重点但不局限于以下内容：词法分析、Thompson算法、子集法构造DFA、DFA最小化；自上而下语法分析、LL(1)文法、预测分析表；自下而上语法分析、短语、直接短语、句柄、识别活前缀的DFA、LR(0)分析表或SLR(1)分析表；语法制导翻译、布尔表达式短路计算、拉链回填。

1 | 填空题总结

19年填空

  1. 识别3型文法描述语言的自动机是有限自动机
  2. 描述含010的01串的正规式是(01)*010(0|1)*
  3. 词法分析器的输入是源程序, 输出是记号流
  4. 自下而上的语法分析中, 需要对输入序列进行规范归约, 反复用产生式的左部替换句型中的句柄, 最终得到开始符号
  5. 在自上而下的语法分析中, 需要消除左递归以避免死循环, 需要提取左因子以避免虚假匹配(陷入回溯)
  6. 设有表达式a * b - c, 将其中的a * b识别为表达式的编译阶段是语法分析
  7. 语义规则的两种表示方式是语法制导定义和翻译方案
  8. 综合属性在语法树上的计算次序和自下而上语法分析形成语法树的过程一致
  9. CFG无法描述语法中的变量声明与引用, 可以在语义规则中通过对符号表的插入, 查找等操作实现
  10. 参数传递的方法有值调用, 引用调用, 复写-恢复, 换名调用, 其中引用调用和复写-恢复要求实参一定是左值
  11. LL(1)分析中, 第一个L表示自左向右扫描输入串, 第二个L表示最左推导, 1表示向前看1个终结符
  12. 常见的中间代码形式有三地址码, 后缀式, 四元式
  13. 程序变量的重复定义会在_阶段被发现

14年填空

  1. 符号表管理和出错处理是编译程序各阶段都涉及到的工作
  2. 在编译器工作过程中, 实现语言关键字大小写不敏感的阶段是词法分析, 分析语言结构的阶段是语法分析
  3. 不确定有限自动机中的有限是指状态的数量是有限的
  4. 程序的语义错误可分为静态语义错误和动态语义错误
  5. 推导的过程可以用一棵树来表示, 被称为分析树
  6. a*-(b+c)的后缀式是abc+-*
  7. 当文法的FIRST集两两不相交时, 该文法对应的句子分析不含有回溯性
  8. 在自上而下的语法分析中, 应先消除文法的间接递归, 再消除文法的直接递归

15年填空

  1. Lex和Yacc是用于生成词法分析器和语法分析器的工具
  2. 描述含010和01串的正规式是(01)*010(0|1)*
  3. 识别3型文法描述语言的自动机是有限自动机
  4. CFG的定义包含有非终结符集合, 终结符集合, 开始符号, 产生式
  5. 自下而上的分析中, 需要对输入序列进行规范归约, 反复用产生式的左部替换句型的句柄, 最终得到开始符号
  6. 文法产生二义性的原因是缺少对文法符号优先级和结合性的规定
  7. 语义规则的两种表示方式是语法制导定义和翻译方案
  8. 综合属性再分析树上的计算次序与自下而上语法分析形成分析树的过程一致
  9. LR分析的语法制导翻译将语义规则放在产生式右部的最右边, LR分析器在执行归约动作时执行语义动作
  10. 赋值语句x = a + (b * c)的后缀式是xabc*+=
  11. CFG无法描述在语法中的变量声明与引用, 可在语义规则中通过对符号表的插入, 查找等操作实现
  12. 参数传递的方法有值调用, 引用调用, 复写-恢复, 换名调用

作业-引言

  1. 一个编译程序中包含的部分除了词法分析, 语法分析, 语义分析, 中间代码生成, 代码优化, 目标代码生成五个部分, 还应包括出错处理和符号表管理
  2. 编译器的各阶段工作中, 中间代码生成和代码优化部分不是每个编译程序都必须的
  3. 词法分析器用于识别单词
  4. 语法分析器可以发现源程序中的语法错误
  5. 解释器处理程序语言时, 大多数采用的是先将程序转化为中间代码, 再解释执行
  6. 高级语言的语言处理程序分为解释程序和编译程序两种, 编译程序有五个阶段, 而解释程序通常缺少目标代码生成和代码优化
  7. 受到具体机器主存容量的限制, 编译程序几个不同阶段的工作往往被组合成遍, 诸阶段的工作往往是穿插进行的
  8. 编译方式和解释方式的根本区别在于是否生成目标代码
  9. 从功能上说, 程序语言的语句大体可分为执行性语句和声明性语句两大类

作业-词法分析

  1. 词法分析器用于识别单词
  2. 三型文法是正规文法
  3. 表示源程序中信息单元的字符序列叫做记号
  4. 有限字母表上有限长度的字符串的集合叫做语言
  5. LEX源程序的三个组成部分: 定义部分, 识别规则部分, 辅助函数部分

作业-语法分析

  1. 一个句型中的最左直接短语称为该句型的句柄
  2. 有限自动机能识别正规文法
  3. 由文法的开始符号S经零步或多步推导产生的文法符号序列是句型
  4. 规范规约是指最右推导的逆过程
  5. 在LR分析法中, 分析栈中存放的状态是识别规范句型活前缀的DFA状态
  6. 设G是一个给定的文法，S是文法的开始符号，如果S*=>x（其中x∈（N∪T)*），则称x是文法的一个句型
  7. 设G是一个给定的文法，S是文法的开始符号，如果S*=>x（其中x∈T*），则称x是文法的一个句子
  8. 在自上而下的语法分析中, 应先消除文法的间接递归, 再消除文法的直接递归
  9. 规范归约是指在移进过程中, 当发现栈顶呈现句柄时, 就用相应产生式的左部符号进行替换
  10. 在LR(0)分析法的名称中, L的含义是从左向右扫描输入串, R的含义是最右推导, 0的含义是向前看0个终结符

作业-语法制导翻译

  1. 中间代码生成时所根据的是语义规则
  2. 编译程序中语法分析器接收以单词为单位的输入
  3. 终结符只具有综合属性, 它由词法分析器提供
  4. 所谓属性文法指的是在上下文无关文法的基础上为每个文法符号（终结符或非终结符）配备若干相关的“值”
  5. 语义规则的描述方法有语法制导定义和翻译方案
  6. 综合属性是用于自下而上传递信息, 继承属性是用于自上而下传递信息
  7. 语法制导翻译既可以用来产生中间代码, 也可用来产生目标指令, 甚至可以用来对输入串进行解释执行
  8. 在语法树中, 一个结点的综合属性的值由其子结点的属性和自身的其他属性值确定, 而继承属性则由该结点的父结点和兄弟结点的某些属性确定

作业-引言

  1. 编译程序在逻辑上由哪几部分组成?

    词法分析, 语法分析, 语义分析, 中间代码生成, 代码优化, 目标代码生成

  2. 编译的前端和后端分别是什么

    前端: 词法分析, 语法分析, 语义分析, 中间代码生成

    后端: 代码优化, 目标代码生成

  3. 什么是编译程序

    编译程序是一种能够把高级语言程序翻译为低级语言程序的程序

  4. 什么是语义

    一种语言的单词符号和语法单位的意义

  5. 什么是程序的遍

    遍就是对源程序或程序的中间结构从头到尾扫描一次, 并做有关的加工处理, 生成新的中间结果或目标程序

作业-词法分析

  1. 简要说明词法分析器的功能

    扫描字符串形式的源程序中的各个字符, 逐个识别出其中的单词, 并将其转换为内部编码形式的单词符号串作为输出

  2. 简要叙述从正规式构造词法分析器的一般方法和过程

    1. 用正规式对模式进行描述
    2. 为每一个正规式构造NFA
    3. 将构造出的NFA转化为等价的DFA(确定化)
    4. 把DFA化为最简形式(最小化)
    5. 将简化后的DFA构造词法分析器

  3. DFA和NFA有何区别?

    1. NFA可以有ε转移, 而DFA没有ε转移
    2. DFA对于每一个状态s和每一个字符a, 最多有一个下一状态, 而NFA可能有多个

  4. NFA的表示

    NFA是一个五元组M = (S, Σ, move, s0, F)

    S是有限状态集合, Σ是输入符号集合, move是状态转换函数, S0是唯一的初态, F是终态集合

作业-语法分析

  1. 最左/右推导和左/右句型的定义

    最左/右推导: 在推导过程中, 每次直接推导均替换句型中最左/右边的非终结符, 称为最左/右推导

    左/右句型: 最左/右推导产生的句型

  2. 自上而下分析的宗旨

    对于给定输入串w, 试图用一切可能的方法, 从文法开始符号S出发, 自上而下, 从左到右地为输入串建立一棵以S为根结点的语法树

  3. 叙述递归下降法

    递归下降法是指对消除左递归和回溯以后的文法的每一个非终结符号, 都根据响应产生式各候选式的结构, 为其编写一个子程序(或函数), 用来识别该非终结符表示的语法范畴, 由于文法的定义是递归的, 因此这些过程也是递归的

  4. 根据程序错误性质可以分为哪几种错误?

    1. 词法错误
    2. 语法错误
    3. 静态语义错误 (如算符作用于不相容的运算对象)
    4. 动态语义错误 (逻辑错误, 如无穷的递归调用)

  5. 自下而上分析的思想

    从输入串开始, 逐步进行「归约」, 直到归约到文法的开始符号; 或者说从语法树的末端开始, 步步向上归约到根结点

  6. 上下文无关文法的表示

    四元组(N, T, P, S)

    N是非终结符集合, T是终结符集合, P是产生式集合, S是开始符号

  7. 句柄是什么

    一个句型的最左直接短语

  8. 活前缀是什么

    右句型的前缀并且不包含句柄之后的符号

作业-语法制导翻译

  1. 语法制导翻译的基本思想是什么?

    基本思想是根据翻译的需要设置文法符号的属性，以描述语法结构的语义

  2. 常用的中间语言种类有哪几种, 使用中间语言的好处有哪些?

    常用的中间语言种类有逆波兰表示、三元式、四元式和树形表示。

    中间代码上起一个编译器前端与后端分水岭的作用，目的是便于编译器的开发移植和代码的优化。

  3. 三元式和四元式的表示

    三元式: (i) (op, arg1, arg2)

    四元式: (op, arg1, arg2, result)

  4. 语义规则

    对于文法的每个产生式都配备了一组属性的计算规则, 称为语义规则

  5. 翻译方案

    用具体属性和运算表示的语义规则称为翻译方案

词法分析

题型1 | 正规式->最小化DFA

方法步骤:

  1. 正规式->NFA (thompson算法)
  2. 由NFA->DFA (子集法)
  3. 最小化DFA
  • 几条基本的Thompson转换:

  • 子集法确定化的方式:
    • 首先求出ε-闭包({0}) = {状态集合}并标上序号A作为A状态
    • 对于每一个状态X和每一个标记a求ε-闭包(smove(X, a)) = {状态集合}并标上序号, 如果状态集合包含终态, 则化俩圈圈, 不包含终态就画一个圈圈
  • 最小化DFA的方式
    • 首先将所有状态分为包含终态的G1和不包含终态的G2
    • 对于G1和G2的每一个转移move(s, a), 假设s和t都在G1中, 如果出现了move(s, a)在G2中, 而move(t, a)在G1中, 则将s和t分裂到两个集合G11和G12中
    • 直到不能再分, 最后将每一个集合作为一个状态, 构造最后的DFA就是最小化DFA

语法分析

题型1 | LL(1)文法构造预测分析表

方法步骤:

  1. 首先消除左递归和左因子, 构造原来文法G的等价的LL(1)文法G’
  2. 求每一个非终结符的FIRST集合和FOLLOW集合
  3. 画出预测分析表M, 第一列是所有非终结符, 第二列是所有终结符 + #
  4. 对于每一个产生式A->α, 对于每一个FIRST(α) = a, 在M[A, a]上填上这个产生式, 如果FIRST(α) = ε, 则找到FOLLOW(A) = b, 在所有M[A, b]上填上产生式A->ε

  • 消除左递归和左因子的方式

    消除左递归先消除间接左递归(展开), 再消除直接左递归

    方法见下面例子

    消除左因子就是提出所有的左因子, 再增加一个产生式

  • 求FIRST集合的方式

    • 对于产生式A->a...加入a到FIRST(A)

    • 对于产生式A->BCDE..首先加入FIRST(B)到FIRST(A), 然后若FIRST(B)包含ε, 就加入FIRST(C)….

  • 求FOLLOW集合的方式

    • 首先加入#到FOLLOW(S) (S为文法开始符号)
    • 对于产生式A->..Ba..将a加入FOLLOW(B)
    • 对于产生式A->..BC..将FIRST(C)除了ε外的全体加入FOLLOW(B)
    • 对于产生式A->..B将FOLLOW(A)全体加入FOLLOW(B)
    • 对于产生式A->..BC 且ε∈FIRST(C)FOLLOW(A)全体加入FOLLOW(B)

注意FIRST集合中包含ε, FOLLOW集合中不包含ε, FOLLOW集中首先要加上#, 构造M[A, a]时是对于每一个A->α中的FIRST(α)而不是FIRST(A)

题型2 | LR(0)分析表和SLR(1)分析表

方法步骤:

  1. 写出拓广文法即在原文法中加入S'->S
  2. 写出识别活前缀的DFA(I0开始标号)
  3. 判断DFA中是否有移进-归约冲突/归约-归约冲突
    • 如果没有, 文法为LR(0)文法, 直接构造LR(0)分析表
    • 如果有, 判断移进项{a1, a2…}, 每一个归约项的FOLLOW集是否相交, 如果不相交就是SLR(1)文法, 构造SLR(1)分析表
  4. SLR(1)分析表的样子: 第一列为所有的DFA状态序号0-n, 第一行分为Action和Goto, Action中包括所有终结符和#, Goto包含所有非终结符
  5. SLR(1)分析表的四种情况:
    1. A->α.aβ: 将Action[k, a] = sj (j为下一状态序号)
    2. A->α.: 将Action[k, a] = rj (a为属于FOLLOW(A)的终结符, j为归约的产生式标号) (注意这里是和LR(0)分析表的唯一区别, LR(0)分析表这里是对Action的所有终结符均记Action[k, a] = rj)
    3. S'->S: 将Action[k, #] = acc
    4. A->α.Bβ: 将Goto[k, B] = j (j为下一状态序号)

注意:

  • 当某个状态的下一状态推出来就是已存在的状态的子集时, 应将边指回到已存在的那个状态
  • Z’不出现在GOTO表中
